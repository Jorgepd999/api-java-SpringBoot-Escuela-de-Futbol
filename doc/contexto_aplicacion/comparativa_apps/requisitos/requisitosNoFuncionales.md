# Requisitos No Funcionales

## RNF-01 Arquitectura
El sistema debe seguir una arquitectura en capas.

## RNF-02 Persistencia
La base de datos utilizada en desarrollo será H2.

## RNF-03 Validación
Los datos de entrada deben validarse antes de persistirse.

## RNF-04 Documentación
La API debe estar documentada automáticamente mediante OpenAPI.

## RNF-05 Portabilidad
El proyecto debe poder ejecutarse mediante DevContainer.

## RNF-06 Mantenibilidad
El código debe estar organizado en paquetes coherentes.

## RNF-07 Escalabilidad
La arquitectura debe permitir migrar fácilmente a base de datos externa.
