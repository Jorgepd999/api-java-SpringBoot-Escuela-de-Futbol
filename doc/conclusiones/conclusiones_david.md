# Análisis DAFO — Footflow

Autor: David Aguilar Guerra
Fecha: 2026-02-20

## Fortalezas

- Base tecnológica sólida (Java + Spring Boot).  
- Documentación y estructura del repo (carpeta `/doc`).  
- Contenerización y CI contemplados (Docker, GitHub Actions).  
- Enfoque práctico: funcionalidad útil para escuelas de fútbol.

## Debilidades

- Alcance ambicioso para el tiempo disponible.  
- Poca experiencia previa en algunas áreas (UX avanzado, despliegue).  
- Pruebas E2E y usabilidad todavía por completar.  
- Recursos limitados (equipo pequeño, posibles costes externos).

## Oportunidades

- Mercado real (academias deportivas) para posibles mejoras o demo.  
- Aprendizaje y visibilidad profesional (DevOps, pruebas, documentación).  
- Posibilidad de añadir features diferenciadoras (chat, asistente IA).  
- Comunidad open-source para apoyo y contribuciones.

## Amenazas

- Cambios en requisitos o alcance que afecten el cronograma.  
- Problemas técnicos con integraciones externas (API IA, servicios).  
- Restricciones legales / privacidad (RGPD) si se usan datos reales.  
- Competencia de soluciones comerciales consolidadas.

## Conclusiones personales

- La documentación y la automatización son clave.  
- Priorizar un MVP claro es esencial para entregar a tiempo.  
- El proyecto tiene potencial real, pero hay que controlar el alcance.

## Próximos pasos recomendados

1. Priorizar y completar el MVP (CRUD, auth, Swagger, Docker).  
2. Añadir pruebas básicas (unitarias + integración) y pipeline estable.  
3. Evaluar i18n y PDF como siguientes mejoras; dejar chat/IA como extras.
